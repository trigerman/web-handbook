Lesson 4: Add Teammates & Work Together

Spin up a shared repo, invite your classmates, collaborate on branches, and merge through pull requests.

Step 1

Create a fresh repository

Head to github.com/new and set up a space for your team.

  • Give the project a clear name everyone recognises.
  • Pick Public for open work or Private for class-only projects.
  • Leave the README unchecked we'll add files from our machines.
Step 2

Invite your teammates

Share access so everyone can push, pull, and review without roadblocks.

  1. Open Settings → Collaborators inside the repo.
  2. Click Add people and search by GitHub username or email.
  3. Send the invite the teammates accept it from their email or GitHub inbox.
Step 3

Clone the repo locally

Grab the SSH URL so your local copy stays in sync with GitHub.

bash 
git clone git@github.com:username/repository-name.git

Swap in your actual GitHub username and repository name before you run the command.

Step 4

Branch for every change

Keep main production-ready by working from feature branches.

bash 
git checkout -b feature-branch-name

Commit your work, then publish the branch so the team can review it:

bash 
git push origin feature-branch-name
Step 5

Open a pull request

Merge through review so teammates can double-check changes before they land in main.

  1. On GitHub, click Compare & pull request after pushing your branch.
  2. Describe what you did and request reviewers.
  3. Address feedback, then merge when everyone approves.

Now you're collaborating like a team

Repo created, invites sent, branches flowing, reviews happening. One more step and your project will ship to the web.
