Lesson 6: Host Your Project on GitHub Pages

Get your site online: push the code, enable Pages, verify the deployment, and show it to the world.

Step 1

Push your project

Stage everything, commit, and push so GitHub has the files it needs to serve.

bash 
git add .
git commit -m "Initial commit"
git push origin main

Make sure main is the branch you use for deployment.

Step 2

Turn on GitHub Pages

Pages will build and host your static site in minutes.

  1. Open Settings → Pages in your repository.
  2. Set Source to Main and /root, then save.
  3. Wait for the green checkmark that confirms the build succeeded.

Your site will be available at https://username.github.io/repository.

Step 3

Confirm it's live

Visit the Pages URL and make sure your homepage loads as expected.

Seeing GitHub's 404 page? Wait a minute and refresh—builds and DNS take a moment to finish. Yes, this tutorial is hosted on GitHub Pages too.
